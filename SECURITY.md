# Security Policy

## Supported Versions

This awesome-data-engineering list is continuously updated. We recommend always using the latest version from the `main` branch.

## Reporting a Vulnerability

While this repository is a curated list and doesn't contain executable code, security concerns can still arise:

### What to Report

- **Malicious links or resources** - Tools or websites that contain malware, phishing attempts, or other security threats
- **Compromised tools** - If a listed tool has been compromised or taken over by malicious actors
- **Misleading information** - Links that redirect to unexpected or harmful content
- **Privacy concerns** - Tools that violate user privacy or contain tracking/surveillance without disclosure

### How to Report

**For Critical Security Issues:**

1. **DO NOT** create a public GitHub issue
2. Email the maintainers directly at the email listed in the repository
3. Include:
   - Description of the security concern
   - The specific tool/link affected
   - Evidence or proof of the vulnerability
   - Suggested remediation

**For Non-Critical Issues:**

- Open a GitHub issue with the label `security`
- Provide details about the concern
- We will review and respond within 48-72 hours

## Security Best Practices for Contributors

When contributing to this list:

1. **Verify Sources** - Only add tools from official, reputable sources
2. **Check HTTPS** - Prefer HTTPS links when available
3. **Avoid Shortened URLs** - Use full, direct URLs to avoid redirect attacks
4. **Research Tools** - Ensure tools have active maintenance and good security practices
5. **Report Suspicious Activity** - If you notice anything unusual, report it

## Automated Security Checks

This repository implements:

- **Link validation** - Weekly automated checks for broken or suspicious links
- **Dependency scanning** - Monitoring for any dependencies (if applicable)
- **Content review** - Regular audits of all listed resources

## Disclosure Policy

- We will acknowledge security reports within 48 hours
- For valid security issues, we will:
  - Remove or update affected content immediately
  - Credit the reporter (unless anonymity is requested)
  - Document the issue in our changelog

## Contact

For security concerns, please open an issue or contact the repository maintainers.

Thank you for helping keep awesome-data-engineering safe and trustworthy!
